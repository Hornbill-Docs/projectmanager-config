# Simple Lists
The Simple Lists provide a way to define different pick lists or combo-box selections that are used through out the Project Manager app. The available options in a pick list or combo-box can also be translated into different languages from within the Simple List editor.

## Project Manager Simple Lists
* **Program Attachment Types**. Define Program Attachment types, default is Image.
* **Program Business Benefit Statuses**. Define the available statuses for each program business benefit.
* **Program Business Benefits**. Define the available business benefits (outcomes) which programs can be assessed against.
* **Program Member Roles**. Define the available program roles which can be assigned to the program members.
* **Project Cost Type**. Define different cost types to reflect the groupings for costs on projects, defaults include General and IT Costs.
* **Project Dependency Priorities**. Define different priorities for project dependancies, these will be available when adding dependancies to projects.
* **Project Dependency Types**. Define the different types of project dependancies, these will be available when adding dependancies to projects.
* **Project Issue Impacts**. Define project issue impacts, these will be available when adding issues to projects.
* **Project Issue Seriousness**. Define project issues seriousness, these will be available when adding issues to projects.
* **Project Member Role**. The Project Member Role defines the available roles that can be selected when adding a stakeholder to a project.
* **Project Member Status**. The Member Statuses list the different statuses that a stakeholder can be in during the life of a project. This can be used to describer when a stakeholder is involved in another project and is currently unavailable for one of their assigned projects.
* **Project Milestone Status**. Define the different statuses that can be used to describe the progress of a Milestone.
* **Project Relationship Type Assets**. Define the different types of available relationships when linking Assets to a project.
* **Project Relationship Type Requests**. Define the different types of available relationships when linking Requests to a project.
* **Project Relationship Type Services**. Define the different types of available relationships when linking Services to a project.
* **Project Risks Matrix Impacts**. Define the different types of available impact levels for project risks.
* **Project Risks Matrix Probabilities**. Define the different types of available probabilities for the project risks.
* **Project Sub Status**. Define any required Sub Statuses which you can then link to Project Statuses.
* **Project Risk Status**. Define the different statuses that can be used to describe the state of a defined risk.
