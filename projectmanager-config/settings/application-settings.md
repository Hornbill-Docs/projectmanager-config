# Advanced Tools & Settings

## Settings

|Name|Description|
|-|---|
|app.com.hornbill.projectmanager.businessProcess.default|The default business process to be used when creating a new project.|
|app.com.hornbill.projectmanager.progressiveCapture.newProject|The Intelligent Capture workflow to be used when creating a new project.|
|app.com.hornbill.projectmanager.project.hud.pin|Setting for pinned/unpinned project workflow progress display.|
|app.com.hornbill.projectmanager.project.stakeholders.addCreatorAsProjectManager|Setting for automatically adding the project creator as the project manager for that project. This Setting can be overridden when automating the raising of a new Project from Hornbill's Integration Bridge (iBridge) and where a Named User or Variable is used to define the Project Manager for the automatically created project. If a named user or variables is not defined in the iBridge input settings, this setting will apply if enabled and the creator will be set as the Project Owner.|
|app.com.hornbill.projectmanager.project.status.preventProjectClosureWithOutstandingItems|Setting for controlling whether or not a project can be closed or canceled whilst there are still active tasks and outstanding milestones - defaulted to On|
|app.com.hornbill.projectmanager.projectTasks.action.defaultVisibility.recordedTime|The default visibility applied when posting to the project task timeline|
|app.com.hornbill.projectmanager.projectTasks.action.defaultVisibility.update|The default visibility applied when posting to the project task timeline|
|app.com.hornbill.projectmanager.milestones.notification.notifyProjectManagerOnCompletion|Setting for controlling whether or not the flag for notifying the project manager on completion is turned on or off by default when creating a new milestone|
|app.com.hornbill.projectmanager.projectTasks.notification.notifyProjectManagerOnCompletion|Setting for controlling whether or not the flag for notifying the project manager on completion is turned on or off by default when creating a new project task.|
